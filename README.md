# 一站式 AI 赋能财富管理全旅程

**人人都是专业财富顾问**

面向理财经理的一站式智能投顾工作台：从**发现客户问题**、**给出专业诊断**，到**一键生成配仓方案**，再到**投后自动触达**，把复杂的专业能力封装成可操作的日常流程。

> 让每位理财经理，都能像资深顾问一样服务客户。

---

## 产品定位


| 维度      | 说明                                    |
| ------- | ------------------------------------- |
| **为谁**  | 一线理财经理、财富经理                           |
| **做什么** | 四笔钱框架下的客户资产检视、智能配仓、投后管理               |
| **价值**  | 缩短从「发现问题」到「给出方案、触达客户」的链路，提升服务专业度与响应速度 |

## 我们解决什么问题

**理财经理的日常，往往卡在三个环节：**

1. **看不清** — 客户资产分散，健康度、偏离点难以一眼识别
2. **配不准** — 模型目标与客户持仓差距大，手工调仓费时且易出错
3. **跟不紧** — 投后预警、回撤、收益异常等信息分散，难以及时生成对客材料并触达经理

**智能资配**将上述环节串成一条可落地的业务闭环，而不是孤立的工具集合。

## 核心能力

### 一、财富洞察 · 先诊断、再行动

- **目标客户盘点**：按场景聚合客户清单，快速锁定需要介入的对象
- **资产诊断**：综合评分、多维能力雷达、四笔钱结构对比，一眼看清「哪里偏了、偏多少」
- **财富健康洞见**：自动汇总结构偏离、业绩异常、投后预警等信号，并给出可执行的诊断结论
- **智能顾问**：基于客户资产与方案上下文的 AI 对话，辅助解读与沟通话术

> 核心理念：**从问题切入，而非一上来就配产品。**

### 二、智能配仓 · 少动手、更专业

- **一键智能配仓**：在模型目标区间内，用尽量少的买卖把持仓调到合理位置
- **因人施策**：根据客户健康标志，自动选择「全账户优化」或「个性化大类处方」等策略
- **方案可编辑**：支持人工微调，修改后实时重算，兼顾算法效率与经理专业判断
- **方案可解读**：自动生成调仓逻辑说明，方便对内说明、对客沟通
- **配仓引擎算法**：基于「区间约束优化」求解，最小交易量优先，标志驱动策略适配

> 核心理念：**算法给出建议，经理保留最终话语权。**

### 三、投后管理 · 接化发、全闭环

- **事件自动识别**：按产品规则日终扫描，识别回撤、收益等异常并写入事件库
- **内容一键生成**：自动产出事件说明、投研分析、对客话术等完整材料包
- **飞书精准触达**：按持有关系，将售后提醒一对一推送给对应客户经理
- **定时 + 手动并行**：支持每晚定时跑批，也保留管理台手动触发，灵活可控

> 核心理念：**投后不是事后补救，而是持续经营的日常动作。**

### 四、智能顾问 · 主副驾驶、智能编排

- **意图智能识别**：自动识别「配仓」「诊断」「解读」「话术」「投后」等意图，精准路由
- **多轮对话闭环**：状态机驱动，从诊断 → 配仓 → 解读 → 话术，全程引导不卡壳
- **工具自动编排**：一键触发配仓引擎、诊断查询、投后事件检索，无需手动跳转
- **方案解读生成**：调用大模型解读调仓逻辑，生成专业、易懂的说明文案
- **客户话术提炼**：自动生成对客沟通话术，支持一键复制或推送飞书

> 核心理念：**副驾驶理解意图与编排工具，主功能区展示数据与执行操作，两者协同。**

## 智能配仓引擎详解

### 核心算法：多约束区间优化

配仓引擎基于「四笔钱框架」，在模型目标区间约束下求解最优调仓方案：


| 输入维度   | 数据来源                     | 约束类型        |
| ------ | ------------------------ | ----------- |
| 客户风险画像 | `portfolio_mapping.yaml` | 决定模型 P1-P5  |
| 当前持仓结构 | 客户资产数据                   | 初始状态        |
| 目标区间   | `model_config.yaml`      | 下限, 基准, 上限] |
| 产品库    | `product_library.yaml`   | 可交易标的       |


### 算法流程

```
┌─────────────────────────────────────────────────────────────────────┐
│                        配仓引擎求解流程                               │
├─────────────────────────────────────────────────────────────────────┤
│  1. 约束建模                                                         │
│     · 解析模型目标区间（固收/权益/现金/另类/保障）                      │
│     · 计算当前持仓占比偏差                                            │
│     · 识别健康标志（回撤/波动/收益/结构偏离）                          │
├─────────────────────────────────────────────────────────────────────┤
│  2. 策略适配                                                         │
│     · 无标志 → 全账户智能优化（最小交易量）                            │
│     · 有标志 → 个性化大类处方（针对性调仓）                            │
│     · 单类优化 → 仅调整指定大类，其他锁定                              │
├─────────────────────────────────────────────────────────────────────┤
│  3. 优化求解                                                         │
│     · 目标函数：最小化买卖金额总和                                    │
│     · 约束条件：各类占比 ∈ [下限, 上限]                               │
│     · 输出：增减仓金额明细 + 调整后结构                                │
├─────────────────────────────────────────────────────────────────────┤
│  4. 方案输出                                                         │
│     · 四笔钱大类汇总（当前/目标/调整）                                 │
│     · 底层产品明细（增减仓金额）                                      │
│     · 调仓逻辑说明（AI 自动生成）                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 核心特性


| 特性          | 说明                           | 价值           |
| ----------- | ---------------------------- | ------------ |
| **区间约束优化**  | 在 下限, 基准, 上限] 区间内求解，而非强制命中基准 | 兼顾风险边界与灵活性   |
| **最小交易量优先** | 用最少买卖达成调仓目标                  | 降低交易成本与操作复杂度 |
| **标志驱动适配**  | 根据健康标志自动选择配仓策略               | 精准响应客户痛点     |
| **人机协同闭环**  | 算法建议 + 经理微调确认                | 专业判断不被黑盒替代   |
| **实时联动重算**  | 人工修改后立即重新求解                  | 支持方案二次调整     |


### 配仓模式对比


| 模式        | 适用场景         | 算法行为         |
| --------- | ------------ | ------------ |
| **全账户优化** | 配置健康、需整体调整   | 全类参与优化，最小交易量 |
| **个性化处方** | 有健康标志、需针对性调整 | 优先调整标志相关大类   |
| **单类优化**  | 仅需调整某一类      | 锁定其他类，仅优化目标类 |
| **人工微调**  | 经理手动修改金额     | 以修改值为约束重新求解  |

## 智能顾问 Agent 编排

### 核心架构：意图路由 + 状态机 + 工具编排

智能顾问基于「Agent 编排框架」，实现多轮对话的意图识别、状态流转与工具调用：

```
┌─────────────────────────────────────────────────────────────────────┐
│                      智能顾问 Agent 编排架构                          │
├─────────────────────────────────────────────────────────────────────┤
│  用户输入                                                            │
│     ↓                                                               │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  意图检测层（Intent Detection）                               │   │

│  ┌─────────────────────────────────────────────────────────────┐   │

│  输出：回复 + 行动卡片 + 旅程更新                                    │

```

### 状态机流转


| 状态            | 触发条件    | 可执行动作                                   | 下一状态          |
| ------------- | ------- | --------------------------------------- | ------------- |
| **idle**      | 无诊断、无方案 | diagnose, rebalance, sop, fullservice   | diagnosed     |
| **diagnosed** | 有诊断、无方案 | rebalance, explain, sop, fullservice    | planned       |
| **planned**   | 有方案、无话术 | explain, generatescript, sop, rebalance | scripted      |
| **scripted**  | 有话术     | sop, copyscript, rebalance              | 保持或回到 planned |


### 意图识别规则


| 意图              | 匹配关键词                 | 处理逻辑          |
| --------------- | --------------------- | ------------- |
| **fullservice** | "帮我服务"、"全流程"、"完整服务"   | 从诊断开始，自动推进到配仓 |
| **rebalance**   | "配仓"、"调仓"、"一键"、"生成方案" | 调用配仓引擎生成方案    |
| **diagnosis**   | "诊断"、"健康度"、"什么问题"     | 获取诊断数据并解读     |
| **explain**     | "解读"、"说明"、"为什么"       | 调用大模型解读方案/诊断  |
| **script**      | "话术"、"沟通"、"对客"        | 调用大模型生成客户沟通话术 |
| **sop**         | "投后"、"事件"、"预警"、"飞书"   | 查询投后事件并生成材料   |
| **nextstep**    | "下一步"、"接下来"、"继续"      | 根据旅程进度推荐下一步   |
| **chat**        | 其他自然语言                | 调用大模型自由对话     |


### 对话策略适配


| 旅程阶段     | 策略风格         | 欢迎语模板                    | 快捷操作      |
| -------- | ------------ | ------------------------ | --------- |
| **新客户**  | friendly     | "首次服务需要先了解客户情况..."       | 诊断、配仓     |
| **回访客户** | efficient    | "上次服务到 {stage}，需要继续吗？"   | 继续、重新诊断   |
| **配仓后**  | supportive   | "配仓方案已就绪，需要解读或生成话术吗？"    | 解读方案、生成话术 |
| **话术后**  | professional | "话术已生成，是否需要推送飞书或查看投后事件？" | 推送飞书、投后跟进 |


### 工具调用示例

```python
# agent_core/advisor_agent.py
class AdvisorAgentService:
    def turn(self, customer_id, message, *, journey, page, diagnosis, plan):
        # 1. 意图检测
        intent = self._detect_intent(message, current_state)

        # 2. 根据意图路由到对应处理函数
        if intent == "full_service":
            reply, tool_results, actions = self._handle_full_service(...)
        elif intent == "rebalance":
            reply, tool_results, actions = self._handle_rebalance(...)
        elif intent == "explain":
            reply, tool_results, actions = self._handle_explain(...)

        # 3. 返回回复 + 行动卡片 + 旅程更新
        return {
            "reply": reply,
            "actions": actions,      # 可执行的下一步操作
            "journey": journey_ctx,  # 旅程进度更新
            "tool_results": tool_results,
        }
```

### 主副驾驶联动

智能顾问（副驾驶）与主功能区实现双向联动：


| 联动场景     | 副驾驶动作                    | 主功能区响应         |
| -------- | ------------------------ | -------------- |
| **配仓完成** | Agent 调用配仓引擎             | 方案自动刷新，展示调整明细  |
| **跳转页面** | Agent 返回 navigate action | 软导航切换，顾问不刷新    |
| **解读方案** | Agent 调用大模型生成解读          | 解读内容展示在顾问面板    |
| **生成话术** | Agent 调用大模型生成话术          | 话术可复制，一键推送飞书   |
| **投后跟进** | Agent 查询投后事件             | 事件列表展示，可批量生成材料 |


**核心理念**：**副驾驶负责理解意图与编排工具，主功能区负责展示数据与执行操作，两者协同闭环。**

## 投后推送说明

### 工作流

1. 规则跑批 — 识别异常事件并写入事件库
2. 批量生成 — 产出事件描述、投研分析、对客话术
3. 飞书推送 — 按持有关系推送给对应客户经理

定时任务可在 **运营管理 → 批量任务** 配置（默认每日 20:00），支持「跑批 → 生成 → 推送」链式自动化。

### 客户经理飞书账号

在 `config/advisor_directory.yaml` 为经理配置 **手机号 / 邮箱 / 工号** 之一即可，无需手工填写 openid：

- 首次推送时自动解析并缓存
- 支持批量对齐与部门同步
