# 智能资配

**项目**：wealthmanagement · **人人都是专业财富顾问**

面向理财经理的一站式智能投顾工作台：从**发现客户问题**、**给出专业诊断**，到**一键生成配仓方案**，再到**投后自动触达**，把复杂的专业能力封装成可操作的日常流程。

> 让每位理财经理，都能像资深顾问一样服务客户。

---

| 维度      | 说明                                    |

| **为谁**  | 一线理财经理、财富经理                           |
| **做什么** | 四笔钱框架下的客户资产检视、智能配仓、投后管理               |
| **价值**  | 缩短从「发现问题」到「给出方案、触达客户」的链路，提升服务专业度与响应速度 |

## 我们解决什么问题

**理财经理的日常，往往卡在三个环节：**

1. **看不清** — 客户资产分散，健康度、偏离点难以一眼识别
2. **配不准** — 模型目标与客户持仓差距大，手工调仓费时且易出错
3. **跟不紧** — 投后预警、回撤、收益异常等信息分散，难以及时生成对客材料并触达经理

**智能资配**将上述环节串成一条可落地的业务闭环，而不是孤立的工具集合。

---

## 核心能力

### 一、客户洞察 · 先诊断、再行动

- **财富盘点**：按场景聚合客户清单，快速锁定需要介入的对象  
- **资产诊断**：综合评分、多维能力雷达、四笔钱结构对比，一眼看清「哪里偏了、偏多少」  
- **财富健康洞见**：自动汇总结构偏离、业绩异常、投后预警等信号，并给出可执行的诊断结论  
- **智能顾问**：基于客户资产与方案上下文的 AI 对话，辅助解读与沟通话术

> 核心理念：**从问题切入，而非一上来就配产品。**

### 二、智能配仓 · 少动手、更专业

- **一键智能配仓**：在模型目标区间内，用尽量少的买卖把持仓调到合理位置  
- **因人施策**：根据客户健康标志，自动选择「全账户优化」或「个性化大类处方」等策略  
- **方案可编辑**：支持人工微调，修改后实时重算，兼顾算法效率与经理专业判断  
- **方案可解读**：自动生成调仓逻辑说明，方便对内说明、对客沟通

> 核心理念：**算法给出建议，经理保留最终话语权。**

### 三、投后管理 · 预警到触达，全自动闭环

- **事件自动识别**：按产品规则日终扫描，识别回撤、收益等异常并写入事件库  
- **内容一键生成**：自动产出事件说明、投研分析、对客话术等完整材料包  
- **飞书精准触达**：按持有关系，将售后提醒一对一推送给对应客户经理  
- **定时 + 手动并行**：支持每晚定时跑批，也保留管理台手动触发，灵活可控

> 核心理念：**投后不是事后补救，而是持续经营的日常动作。**

## 业务闭环

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  财富盘点   │ ──→ │  资产诊断   │ ──→ │  智能配仓   │ ──→ │  方案落地   │
│  谁需要关注 │     │  问题在哪   │     │  怎么调     │     │  对客沟通   │
└─────────────┘     └──────┬──────┘     └─────────────┘     └─────────────┘
                           │
                    投后预警回灌
                           ↓
                    ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
                    │  事件识别   │ ──→ │  内容生成   │ ──→ │  飞书推送   │
                    │  规则跑批   │     │  话术材料   │     │  经理触达   │
                    └─────────────┘     └─────────────┘     └─────────────┘
```

**一条链路，三个价值：** 看得清 → 配得准 → 跟得紧。

## 快速启动

```bash
cd wealthmanagement
python3 -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env               # 按需填入大模型 / 飞书凭证
python main.py
```

服务默认监听 **[http://127.0.0.1:8000](http://127.0.0.1:8000)**。

### 环境变量（`.env`）


| 变量                  | 说明                        |
| ------------------- | ------------------------- |
| `LLM_API_KEY`       | 大模型 API Key（OpenAI 兼容；可选） |
| `FEISHU_APP_ID`     | 飞书应用 App ID（投后推送）         |
| `FEISHU_APP_SECRET` | 飞书应用 App Secret           |


未配置大模型时，配仓与规则解读仍可用；未配置飞书时，SOP 内容生成可用，推送不可用。

## 配置链路（智能资配）

```
客户风险等级
    ↓
投资组合偏好 / 可承受亏损
    ↓
资产配置模型（P1–P5）
    ↓
五类资产目标区间 [下限, 基准, 上限]
    ↓
四笔钱 / 资产类型阈值
    ↓
智能配仓引擎
```

模型保存时校验：四笔钱基准值之和须等于 100%（容差 0.01%）。

## 投后推送说明

### 工作流

1. 规则跑批 — 识别异常事件并写入事件库
2. 批量生成 — 产出事件描述、投研分析、对客话术
3. 飞书推送 — 按持有关系推送给对应客户经理

定时任务可在 **运营管理 → 批量任务** 配置（默认每日 20:00），支持「跑批 → 生成 → 推送」链式自动化。

### 客户经理飞书账号

在 `config/advisor_directory.yaml` 为经理配置 **手机号 / 邮箱 / 工号** 之一即可，无需手工填写 open_id：

- 首次推送时自动解析并缓存  
- 支持批量对齐与部门同步
